@startuml                                                                                                                                 
  title xx拒单功能时序图                                                                                                                  
                                                                                                                                              
  actor 服务人员                                                                                                                              
  participant xx                                                                                                                          
  participant 履约服务                                                                                                                        
  participant 派单服务                                                                                                                        
                                                                                                                                              
  == 第一步：触发拒单流程 ==                                                                                                                  
                                                                                                                                              
  服务人员 -> xx : 点击"拒绝接单"按钮                                                                                                     
  xx -> xx : 捕获点击事件
  xx -> xx : 显示拒单原因弹窗                                                                                                         
                                                                                                                                              
  == 第二步：填写拒单原因 ==                                                                                                                  
                                                                                                                                              
  loop [输入验证（每次输入）]                                                                                                                 
    服务人员 -> xx : 输入拒单原因
    xx -> xx : 实时验证字数(0 < 长度 ≤ 100)                                                                                           
    xx -> xx : 更新确认按钮状态（禁用/启用）                                                                                          
  end                                                                                                                                         
                                                                                                                                              
  服务人员 -> xx : 点击"确认"按钮                                                                                                         
                                                                                                                                              
  == 第三步：调用履约服务 ==                                                                                                                  
                  
  note right of 履约服务                                                                                                                      
    请求参数:     
    reason: "拒单原因"                                                                                                                        
    serviceOrderId: "xxx"                                                                                                                     
    eventType: 3                                                                                                                              
  end note                                                                                                                                    
                                                                                                                                              
  xx -> 履约服务 : POST: /fulfillment/event/order                                                                                         
  履约服务 -> 履约服务 : 1. 鉴权认证\n2. 校验服务单状态\n3. 更新服务单和记录单状态为"已取消"
  履约服务 -> 派单服务 : 通知订单回退到派单池                                                                                                 
                  
  note right of 派单服务                                                                                                                      
    参数:         
    serviceOrderId: "服务单"                                                                                                                  
    reason: "拒单原因"                                                                                                                        
    techId: "原服务人员ID"                                                                                                                    
  end note                                                                                                                                    
                                                                                                                                              
  派单服务 -> 派单服务 : 1. 将订单重新加入派单池\n2. 更新预约单状态为"已接单待派单"\n3. 避免再次派给同一服务人员                              
                                                                                                                                              
  履约服务 -> xx : 200 OK { success: true, message: "拒单成功" }                                                                          
                  
  == 第四步：更新界面 ==                                                                                                                      
                  
  xx -> xx : 1. 关闭弹窗\n2. 更新订单列表状态为"已取消"\n3. 从列表中移除或标记该订单\n4. 显示Toast提示                                
  xx -> 服务人员 : 显示"拒单成功，已帮您取消该订单"
                                                                                                                                              
  @enduml
