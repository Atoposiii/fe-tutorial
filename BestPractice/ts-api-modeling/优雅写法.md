# JavaScript 日常开发优雅写法指南（工程实践版）

> 核心原则：**清晰 > 简洁 > 炫技**

---

# ✅ 一、推荐使用的优雅写法

这些写法兼顾：**可读性 + 稳定性 + 团队协作**

---

## 1️⃣ 默认值：Nullish Coalescing（推荐）
```js
const count = value ?? 0;
````

* 仅在 `null / undefined` 时生效
* 不会误伤 `0 / "" / false`

---

## 2️⃣ 可选链：安全访问

```js
const city = user?.address?.city;
```

* 避免多层 `if`
* 防止运行时报错

---

## 3️⃣ 解构赋值

```js
const { name, age } = user;
```

* 减少重复访问
* 提高语义清晰度

---

## 4️⃣ 展开运算符（不可变数据）

```js
const newObj = { ...obj, age: 18 };
const newArr = [...arr];
```

* 推荐用于状态管理（React/Vue）
* 避免修改原数据

---

## 5️⃣ 模板字符串

```js
const msg = `Hello ${name}`;
```

* 替代字符串拼接
* 更直观

---

## 6️⃣ 条件执行（简单逻辑）

```js
isLogin && fetchData();
```

* 表达“条件成立才执行”

---

## 7️⃣ 显式类型转换（推荐）

```js
Number(value)
String(value)
Boolean(value)
```

* 可读性强
* 避免隐式转换带来的歧义

---

## 8️⃣ 数组函数式操作

```js
arr.map(x => x * 2);
arr.filter(x => x > 0);
arr.find(x => x.id === 1);
```

* 表达意图清晰
* 避免副作用

---

## 9️⃣ includes 替代 indexOf

```js
arr.includes(3);
```

* 语义更明确

---

## 🔟 提前 return（减少嵌套）

```js
if (!user) return;

doSomething();
```

* 降低代码复杂度
* 提升可读性

---

# ⚠️ 二、不建议使用的写法（避雷）

这些写法虽然“短”，但不适合工程实践

---

## ❌ 1️⃣ 一元 `+`

```js
+value
```

问题：

* 可读性差

推荐：

```js
Number(value)
```

---

## ❌ 2️⃣ `~~` / `| 0`

```js
~~value
value | 0
```

问题：

* 32 位整数限制
* 隐晦难懂

推荐：

```js
Math.floor(value)
```

---

## ❌ 3️⃣ 字符串拼接技巧

```js
value + ""
```

问题：

* 不直观

推荐：

```js
String(value)
```

---

## ❌ 4️⃣ 滥用 `||` 作为默认值

```js
const count = value || 10;
```

问题：

* `0`、`""` 会被覆盖

推荐：

```js
const count = value ?? 10;
```

---

## ❌ 5️⃣ `!!value`

```js
!!value
```

问题：

* 可读性一般

推荐：

```js
Boolean(value)
```

---

## ❌ 6️⃣ 复杂短路逻辑

```js
a && b && doSomething();
```

问题：

* 可读性差

---

## ❌ 7️⃣ 使用 `==`

```js
[] == false
```

问题：

* 隐式转换复杂
* 易出 bug

推荐：

```js
===
```

---

## ❌ 8️⃣ 链式三元表达式

```js
a ? b : c ? d : e;
```

问题：

* 难以阅读

---

## ❌ 9️⃣ 过度压缩逻辑

```js
const x = a && b || c;
```

问题：

* 优先级不清晰
* 容易出错

---

# 🎯 三、编码原则（比语法更重要）

---

## ✅ 1. 代码是写给人看的

```js
Number(price)   // 推荐
+price          // 不推荐
```

---

## ✅ 2. 避免隐式魔法

```js
Boolean(value)  // 推荐
!!value         // 次优
```

---

## ✅ 3. 一致性优先

* 团队统一风格 > 个人习惯

---

## ✅ 4. 语义优先

```js
const isValid = Boolean(value);
```

---

## ✅ 5. 降低嵌套

```js
if (!data) return;
```

---

# 🧾 四、推荐写法速查表

## ✔ 推荐

* `??`（默认值）
* `?.`（安全访问）
* 解构赋值
* 展开运算符
* 模板字符串
* `Number / String / Boolean`
* `map / filter / find`
* 提前 return

---

## ❌ 避免

* `+value`
* `~~value`
* `value + ""`
* `||`（默认值场景）
* `!!value`
* `==`
* 复杂短路逻辑
* 链式三元表达式

---

# 🚀 总结

> 好代码不是“最短的”，而是“最容易被理解的”。
